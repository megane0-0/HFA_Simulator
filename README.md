# 視野検査シミュレーター Web版

ハンフリー自動視野計（24-2・76点）の検査結果をもとに、患者の見え方をカメラ映像で再現するWebアプリです。iOS版（Swift）からの移植版で、単一の `index.html` のみで動作します。

**【重要】本アプリは医療機器ではありません。** 理解促進・教育目的のツールです。データはブラウザ内でのみ処理され、外部送信されません。

## 試し方

### 方法1: ローカルで試す

`index.html` をブラウザで直接開くと、視野データ入力と補間プレビューは動作します。ただし**カメラ機能は `file://` では動かない**（ブラウザのセキュリティ制限）ため、カメラも試す場合はローカルサーバーを起動してください。

```bash
# index.htmlのあるフォルダで実行
python3 -m http.server 8000
```

ブラウザで http://localhost:8000 を開きます（localhostはカメラ利用可）。

### 方法2: GitHub Pagesで公開する（スマホでも試せる）

1. GitHubで新しいリポジトリを作成（例: `hfa-simulator`、Public）
2. `index.html` と `README.md` をリポジトリのルートにpush

```bash
cd HFA-simulator-web
git init
git add index.html README.md
git commit -m "視野検査シミュレーター Web版"
git branch -M main
git remote add origin https://github.com/<ユーザー名>/hfa-simulator.git
git push -u origin main
```

3. リポジトリの **Settings → Pages** を開き、Source を「Deploy from a branch」、Branch を `main` / `(root)` にして Save
4. 1〜2分後に `https://<ユーザー名>.github.io/hfa-simulator/` で公開されます

GitHub PagesはHTTPSなので、スマホのブラウザからこのURLを開けばカメラ機能もそのまま使えます（初回にカメラ許可を求められます）。

## 使い方

1. **① 視野データ入力**: 欠損レベル（0〜4）を選び、76点の測定点をタップまたはドラッグで塗る。プリセット（正常・求心性視野狭窄・中心暗点・上半視野欠損）も利用可。入力内容は自動保存されます
2. **② カメラシミュレーション**: 「カメラ開始」→ 視野半径30度に対応する円の内側に視野欠損フィルターがリアルタイム適用され、円の外側は黒く表示される。「撮影して保存」でPNGとしてダウンロード

## 画角設定（視野角キャリブレーション）

シミュレーション円（視野半径30度）を実際の視野角に対応させるため、「カメラ映像の短辺が視野何度に相当するか」を選択できます。

| 設定 | 焦点距離（35mm判換算） | 短辺の視野角 |
|---|---|---|
| 標準（既定値） | − | 30度（仮定） |
| iPhone 15 / 16 / 17 / 16e / Air | 26mm | 約43度 |
| iPhone 15 / 16 / 17 Pro・Pro Max | 24mm | 約46度 |
| Google Pixel 9 / 10 シリーズ | 25mm | 約44度 |
| Galaxy S24 / S25 シリーズ | 23mm | 約47度 |
| Xperia 1 VII | 24mm | 約46度 |

いずれもメインカメラ（1×）・16:9動画時の値です。

換算方法: 35mm判換算焦点距離 f に対し、16:9動画の短辺視野角 = 2·atan(10.125/f)。ブラウザが実際に使うカメラ・クロップにより多少ずれる可能性があります。他機種のプリセットは同式で追加できます（`index.html` 内 `fovSelect` の option を編集）。

## 視野欠損レベル

| レベル | dB値範囲 | ピクセル化 | ブラー | 表現 |
|---|---|---|---|---|
| 0 | 25dB以上 | なし | なし | 正常・元画像そのまま |
| 1 | 15〜24dB | 8 | 4px | 軽度のぼやけ |
| 2 | 5〜14dB | 14 | 8px | 中等度のぼやけ |
| 3 | 0〜5dB | 32 | 24px + 35%減光 | 形はほぼ判別不能・色調のみ |
| 4 | 0dB以下 | − | − | 真っ暗（完全遮蔽・不透明度強化済み） |

## iOS版からの主な変更点

- **プラットフォーム**: Swift/SwiftUI → 単一HTML（依存ライブラリなし）
- **フィルター処理の修正**: iOS版のリアルタイム処理は「レベル2ブラーを全域に一律適用」に簡略化されていたが、Web版は4レベルそれぞれの強度でマスク合成する本来の仕様を実装
- **補間グリッド**: 32×32 → 64×64（境界がより滑らか）。補間はIDW（逆距離加重、power=2）で同一
- **OCR機能**: 省略（手動入力に一本化）。将来Tesseract.jsで追加可能
- **データ保存**: 写真アルバム → PNGダウンロード。視野データはlocalStorageに自動保存

## 動作環境

- Chrome / Edge / Safari 18以降 推奨（Safari 17以前はブラー近似表示）
- カメラ機能はHTTPSまたはlocalhostでのみ動作
