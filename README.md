# 視野検査シミュレーター Web版

ハンフリー自動視野計（24-2・76点）の検査結果をもとに、患者の見え方をカメラ映像で再現するWebアプリです。iOS版（Swift）からの移植版で、単一の `index.html` で動作します（OCR機能を使う場合のみTesseract.jsをCDNから読み込みます）。

**【重要】本アプリは医療機器ではありません。** 理解促進・教育目的のツールです。データはブラウザ内でのみ処理され、外部送信されません。

## 試し方

### 方法1: ローカルで試す

`index.html` をブラウザで直接開くと、視野データ入力と補間プレビューは動作します。ただし**カメラ機能は `file://` では動かない**（ブラウザのセキュリティ制限）ため、カメラも試す場合はローカルサーバーを起動してください。

```bash
# index.htmlのあるフォルダで実行
python3 -m http.server 8000
```

ブラウザで http://localhost:8000 を開きます（localhostはカメラ利用可）。

### 方法2: GitHub Pagesで公開する（スマホでも試せる）

1. GitHubで新しいリポジトリを作成（例: `hfa-simulator`、Public）
2. `index.html` と `README.md` をリポジトリのルートにpush

```bash
cd HFA-simulator-web
git init
git add index.html README.md
git commit -m "視野検査シミュレーター Web版"
git branch -M main
git remote add origin https://github.com/<ユーザー名>/hfa-simulator.git
git push -u origin main
```

3. リポジトリの **Settings → Pages** を開き、Source を「Deploy from a branch」、Branch を `main` / `(root)` にして Save
4. 1〜2分後に `https://<ユーザー名>.github.io/hfa-simulator/` で公開されます

GitHub PagesはHTTPSなので、スマホのブラウザからこのURLを開けばカメラ機能もそのまま使えます（初回にカメラ許可を求められます）。

## 使い方

### ① 視野データ入力

1. **眼の選択**: 「右眼（OD）」「左眼（OS）」タブで入力する眼を切り替えます。片眼のみの検査結果しかない場合は「この眼をもう一方へコピー」で両眼を同じデータにできます
2. **レベルで塗る**: 欠損レベル（0〜4）を選び、76点の測定点をタップまたはドラッグで塗ります
3. **dB値で塗る**: 検査結果のdB値を忠実に再現したい場合は、数値（0〜40dB）を入力して「この値で塗る」を選択してから点を塗ります
4. **プリセット**: 正常視野・求心性視野狭窄・中心暗点・上半視野欠損の4種を選択中の眼に適用できます
5. 入力内容は自動保存されます（ブラウザのlocalStorage）

### ② カメラシミュレーション

「カメラ開始」を押すと、視野欠損フィルターが視野半径30度に対応する円の内側にリアルタイム適用されます。円の外側は黒く表示されます。「撮影して保存」でPNGとしてダウンロードできます。

画面上部のボタンで**表示する眼**を切り替えられます（選択は自動保存されます）。

| 表示 | 内容 |
|---|---|
| 右眼（OD） | 右眼の入力データのみでシミュレーション |
| 左眼（OS） | 左眼の入力データのみでシミュレーション |
| 両眼（better eye・既定値） | 各測定点で左右の感度が良い方の値を採用した両眼視モデル |

補間マッププレビュー（入力タブ）もここで選択した眼に連動します。

### ③ OCR読み取り（HFA結果紙からの自動入力）

HFA（24-2）の**数値プロット**（dB値が並んだ図）を撮影またはアップロードすると、数値を自動認識して視野データに反映できます。

1. 反映先の眼（右眼／左眼）を選ぶ
2. 「画像をアップロード」または「カメラで撮影」で結果紙の画像を読み込む
3. 数値プロットの範囲を画像上でドラッグして枠選択（任意だが精度が上がる）
4. 「OCR解析」を押す（初回はTesseract.jsのダウンロードに時間がかかります）
5. 検出された数値が青い丸で画像上に表示される。問題なければ「①入力タブに反映して確認」を押す
6. ①入力タブに切り替わるので、**必ず値を確認・修正**する

認識した数値は、検出位置を76点の測定点へ最近傍でマッピングし、割り当てられなかった点は最も近い割当済みの点の値で補完します。OCRの精度には限界がある（照明・傾き・印刷品質に依存）ため、**この機能は下書きの自動入力であり、最終的な値は必ず目視で確認してください**。OCR処理はすべてブラウザ内（端末上）で行われ、画像が外部に送信されることはありません。

## 画角設定（視野角キャリブレーション）

シミュレーション円（視野半径30度）を実際の視野角に対応させるため、「カメラ映像の短辺が視野何度に相当するか」を選択できます。

| 設定 | 焦点距離（35mm判換算） | 短辺の視野角 |
|---|---|---|
| 標準（既定値） | − | 30度（仮定） |
| iPhone 15 / 16 / 17 / 16e / Air | 26mm | 約43度 |
| iPhone 15 / 16 / 17 Pro・Pro Max | 24mm | 約46度 |
| Google Pixel 9 / 10 シリーズ | 25mm | 約44度 |
| Galaxy S24 / S25 シリーズ | 23mm | 約47度 |
| Xperia 1 VII | 24mm | 約46度 |

いずれもメインカメラ（1×）・16:9動画時の値です。

換算方法: 35mm判換算焦点距離 f に対し、16:9動画の短辺視野角 = 2·atan(10.125/f)。ブラウザが実際に使うカメラ・クロップにより多少ずれる可能性があります。他機種のプリセットは同式で追加できます（`index.html` 内 `fovSelect` の option を編集）。

## 視野欠損レベル

| レベル | dB値範囲 | ピクセル化 | ブラー | 表現 |
|---|---|---|---|---|
| 0 | 25dB以上 | なし | なし | 正常・元画像そのまま |
| 1 | 15〜24dB | 8 | 4px | 軽度のぼやけ |
| 2 | 5〜14dB | 14 | 8px | 中等度のぼやけ |
| 3 | 1〜4dB | 32 | 24px + 35%減光 | 形はほぼ判別不能・色調のみ |
| 4 | 0dB以下 | − | − | 真っ暗（完全遮蔽・不透明度強化済み） |

レベルボタンで塗った場合の代表値は 30 / 20 / 10 / 3 / 0 dB です。dB直接入力では0〜40dBの任意の値を設定できます。

## データの扱い

- 視野データは左右眼別にブラウザのlocalStorage（キー `hfa-sim-data-v2`）へ自動保存されます
- 旧形式（`hfa-sim-data-v1`、両眼まとめて入力）のデータがある場合は、初回起動時に左右両眼へ同じ値として自動移行されます
- データはすべて端末内で処理され、外部に送信されることはありません

## iOS版からの主な変更点

- **プラットフォーム**: Swift/SwiftUI → 単一HTML（依存ライブラリなし）
- **フィルター処理の修正**: iOS版のリアルタイム処理は「レベル2ブラーを全域に一律適用」に簡略化されていたが、Web版は4レベルそれぞれの強度でマスク合成する本来の仕様を実装
- **補間グリッド**: 32×32 → 64×64（境界がより滑らか）
- **補間方式の改良**（Web版で変更）: IDW（逆距離加重、power=2）を全点参照から**近傍4点限定**に変更し、中心(0,0)に最内4点の平均値を持つ**仮想中心点**を追加。iOS版・Web版初期の全点IDWでは遠方の正常点に引っ張られ、暗点内部が「測定点直上だけ暗いまだら」になっていたが、この変更で0dB領域が面として暗くなる（例: 中心暗点プリセットでは最内4点の内側が完全に黒く抜ける）
- **左右眼別入力**（Web版で追加）: 右眼・左眼を個別に入力。カメラ画面で右眼/左眼/両眼（better eye採用）を切替可能
- **dB値直接入力**（Web版で追加）: 検査結果のdB値をそのまま入力可能
- **OCR機能**: Web版で実装（③OCR読み取りタブ）。Tesseract.jsをCDNから遅延ロードし、数値プロットを認識して76点へ半自動マッピング。結果は①入力タブで確認・修正する運用
- **データ保存**: 写真アルバム → PNGダウンロード。視野データはlocalStorageに自動保存

## 動作環境

- Chrome / Edge / Safari 18以降 推奨（Safari 17以前はブラー近似表示）
- カメラ機能はHTTPSまたはlocalhostでのみ動作
- OCR機能はTesseract.jsをCDN（cdnjs）から読み込むため、初回利用時はネット接続が必要
